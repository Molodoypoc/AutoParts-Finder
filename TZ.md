Техническое задание на проект: "AutoParts Finder"
1. Цель проекта
    Создание системы, позволяющей владельцам подержанных европейских автомобилей быстро и удобно находить аналоги запчастей по VIN-номеру и OEM-кодам. Система решает проблему высокой стоимости оригинальных запчастей, предлагая пользователям доступные альтернативы и анализируя информацию о совместимости компонентов.
2. Роли пользователей
    Гость: Просмотр каталогов запчастей, доступ к общим характеристикам и рекомендациям.
    Зарегистрированный пользователь: Возможность поиска запчастей по VIN и OEM-кодам, сохранение избранных запчастей, получение уведомлений о скидках и акциях.
    Администратор: Управление данными запчастей, модерация пользовательского контента, отчетность по использованию системы.
3. Модели данных (сущности)
    Part (Запчасть): Основная модель для хранения информации о запчастях.
    Поля: id, name, vin, oem_code, description, compatibility_info, price.
    User (Пользователь): Модель для хранения данных о пользователях системы.
    Поля: id, username, password, email, role, favorites (список избранных запчастей).
    Compatibility (Совместимость): Модель для анализа совместимости запчастей.
    Поля: id, part_id (ссылка на запчасть), compatible_parts (список идентификаторов совместимых запчастей), notes (заметки о совместимости).
4. Ключевой функционал (User Stories)
    Поиск запчастей: Пользователь вводит VIN-номер или OEM-код, система отображает соответствующие аналоги и оригинальные запчасти.
    Сохранение избранного: Пользователь добавляет запчасть в избранное для быстрого доступа к ней в будущем.
    Анализ совместимости: Пользователь видит, какие запчасти совместимы, а также дополнительные заметки о возможности установки.
    Администрирование системы: Администратор добавляет или обновляет информацию о запчастях, управляет пользователями и их правами доступа.
5. Внешние интеграции / Аналитика
    Интеграция: Использование внешнего API для получения информации о запчастях и их ценах. Возможное использование API производителей или коммерческих баз данных о запчастях.
    Аналитика: Применение библиотеки Pandas для анализа данных о продаже запчастей и совместимости, создание отчетов о трендах в запросах пользователей.
